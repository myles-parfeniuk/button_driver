Button\+Driver is a C++ based component written for esp-\/idf version 4.\+0, intended to simplify the use of push-\/buttons and tactile switches with the esp-\/32. ~\newline
 It allows for the creation of \doxylink{class_button}{Button} objects which automatically detect user input from externally connected tactile switches or push-\/buttons. ~\newline
 Call-\/back functions can be registered to button objects and used to handle detected user input. ~\newline

This is intended to be a quick-\/guide, api documentation generated with doxygen can be found in the documentation directory of the master repo.\hypertarget{md__d_1_2development_2esp_2projects_2idf__4__0_2data__control_2components_2button__driver_2_r_e_a_d_m_e_autotoc_md5}{}\doxysubsubsection{\texorpdfstring{Initializing Button Object}{Initializing Button Object}}\label{md__d_1_2development_2esp_2projects_2idf__4__0_2data__control_2components_2button__driver_2_r_e_a_d_m_e_autotoc_md5}
To initialize a button object, first initialize and configure a button\+\_\+conf\+\_\+t struct with the desired settings, then pass it into the \doxylink{class_button}{Button} constructor. ~\newline


The settings available within a button\+\_\+conf\+\_\+t struct\+: ~\newline

\begin{DoxyItemize}
\item {\bfseries{gpio\+\_\+num}} — The GPIO number associated with the button, must not be initialized as GPIO\+\_\+\+NUM\+\_\+\+NC ~\newline

\item {\bfseries{active\+\_\+lo}} — Set to true if the button is active low (falling edge trigger), cannot be true if active\+\_\+hi is also true ~\newline

\item {\bfseries{active\+\_\+hi}} — Set to true if the button is active high (rising edge trigger), cannot be true if active\+\_\+lo is also true ~\newline

\item {\bfseries{pull\+\_\+en}} — Set to true if internal pullup/pulldown resistor is enabled for button gpio pin, set to false if external resistors are used ~\newline

\item {\bfseries{long\+\_\+press\+\_\+evt\+\_\+time}} — long-\/press event generation time in microseconds (us) if the button is held for longer than (long\+\_\+press\+\_\+evt\+\_\+time+25ms) a long-\/press event is generated, if it is released before (long\+\_\+press\+\_\+evt\+\_\+time+25ms) elapses, a quick-\/press event is generated instead suggested time of 300000us, must be between 10000us and 5000000us ~\newline

\item {\bfseries{held\+\_\+event\+\_\+evt\+\_\+time}} — held event generation time in microseconds (us), if a long press event has already occurred and the button is still being held, held events will be generated every held\+\_\+evt\+\_\+time elapses, suggested time of 200000us, must be between 10000us and 5000000us
\end{DoxyItemize}

If the button\+\_\+conf\+\_\+t struct is not initialized correctly the \doxylink{class_button}{Button} constructor will output an error related to the issue in the terminal and dump a stack trace. ~\newline

After being initialized, a \doxylink{class_button}{Button} object will automatically detect any user input and generate an event.

These events come in four flavors\+:


\begin{DoxyEnumerate}
\item {\bfseries{quick-\/press\+:}} ~\newline
 This event indicates the switch was momentarily pressed. This event is generated when the push-\/button is pressed \& then released before (25ms + long\+\_\+press\+\_\+evt\+\_\+time) has elapsed.
\item {\bfseries{long-\/press\+:}} ~\newline
 This event indicates the switch was pressed and held. This event is generated when the push-\/button is pressed \& not released after (25ms + long\+\_\+press\+\_\+evt\+\_\+time) has elapsed.
\item {\bfseries{held\+:}} ~\newline
 This event indicates a long-\/press event has already occurred, and the switch is still being held. This event is generated every time held\+\_\+evt\+\_\+time elapses, after a long-\/press event, until the switch is released.
\item {\bfseries{released\+:}} ~\newline
 This event indicates the switch has been released. This event is generated if the switch is released any time after a long\+\_\+press event has occurred.
\end{DoxyEnumerate}\hypertarget{md__d_1_2development_2esp_2projects_2idf__4__0_2data__control_2components_2button__driver_2_r_e_a_d_m_e_autotoc_md7}{}\doxysubsubsection{\texorpdfstring{Handling Button Events}{Handling Button Events}}\label{md__d_1_2development_2esp_2projects_2idf__4__0_2data__control_2components_2button__driver_2_r_e_a_d_m_e_autotoc_md7}
In order to be notified when a button-\/event has occurred, a call-\/back function (or multiple) can be registered with the button by calling the follow() method on its event member. ~\newline


As many call-\/backs as desired can be registered to a button using follow(). When a button event occurs, any call-\/backs registered to the respective button will be called in the order they were registered— this means whichever call-\/back was registered first has highest priority. ~\newline


It is recommended to initialize the call-\/back functions as lambda-\/functions for easy readability. ~\newline
 Any call-\/back function registered with follow() must take the form\+: ~\newline
